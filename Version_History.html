Version 7.0

Major SeedVR2 Improvements - Scene Split, RAM/VRAM Optimization & Enhanced Batch Processing

v7.0

SeedVR2 Performance & Stability Update

30 July 2025

Version 7.0 brings significant improvements to SeedVR2 upscaling with enhanced scene split processing, fixed memory leaks, improved single image pipeline, and comprehensive batch processing enhancements. This release focuses on stability, performance, and VRAM optimization for better user experience.

Major Improvements: Scene split processing now fully working with SeedVR2, RAM/VRAM leakage fixed, single image pipeline perfected, and batch processing significantly enhanced.

Core SeedVR2 Improvements

  • Scene Split Processing: Scene split based processing now fully working with SeedVR2 and is recommended for optimal results
  • Automatic Resolution Determination: Resolution detection and setting mechanism fixed for accurate upscaling
  • Single Image Pipeline: Single image processing pipeline fixed and working perfectly
  • Preset System: Preset save and load functionality fixed for SeedVR2 configurations

Memory Optimization

  • RAM/VRAM Leak Fixed: Major memory leakage issues resolved with significantly improved VRAM usage patterns
  • VRAM Usage Optimization: 24 GB GPU can now upscale single images to 2560x2560px (6.55 megapixels) in under 30 seconds
  • Video VRAM Management: 24 GB GPU can upscale videos to 1280x720px with batch size 9 (VAE decoding uses 23.5 GB peak)
  • Batch Size Optimization: Due to model restrictions, batch size can only increase by 4 (5, 9, 13, 17, etc.)

Batch Processing Enhancements

  • Batch Image Upscaling: Significantly improved with bug fixes and performance optimizations
  • Settings Synchronization: Batch image upscaling now uses settings from Single Image tab for consistency
  • Multi-Model Support: Batch upscale now works with both SeedVR2 and Image Upscalers like 2xLiveActionV1_SPAN_490000.pth
  • Quality Recommendation: SeedVR2 upscale recommended as it provides SUPIR-level quality or better depending on task

Known Limitations

  • Block Swapping: Currently not working and temporarily disabled until tiled VAE encoding/decoding arrives
  • Tiled VAE Decoding: Not yet implemented but coming soon for significant VRAM reduction
  • FP8 Precision Models: Not useful yet pending further optimization
  • SeedVR2 3B FP16 Model: Has VRAM doubling issue - do not use until fixed

Working Models & Recommendations

  • SeedVR2 7B FP16 Model: Working perfectly and recommended for best quality
  • Testing Presets: Start testing with SeedVR2_Image and SeedVR2_Video presets for optimal results
  • VRAM Management: If out of VRAM, reduce max resolution from Resolution & Scene Split settings
  • Performance Testing: RTX 3090 Ti (24GB) confirmed for all specified performance benchmarks

Additional Fixes

  • Batch Size Slider: Fixed to properly increment by 4 as required by model architecture
  • Performance Improvements: Numerous other bug fixes and performance optimizations throughout the codebase
Coming Soon: Tiled VAE implementation will significantly reduce VRAM usage, allowing larger upscale sizes with bigger batch sizes.
# Recommended workflow: 1. Use SeedVR2 7B FP16 model 2. Enable scene split processing 3. Start with SeedVR2_Image or SeedVR2_Video presets 4. Adjust max resolution if VRAM issues occur 5. Use batch sizes of 5, 9, 13, 17 etc.

Version 4.0

Critical Face Restoration Fixes & Enhanced Session Management

v4.0

Critical Face Restoration Fixes & Enhanced Session Management

21 July 2025

Version 4.0 addresses a critical bug in face restoration where only 3 frames were being output instead of the complete video (e.g., 1,257 frames → 3 frames). This release introduces organized session-based output management, live frame saving, and comprehensive improvements to the face restoration workflow to ensure reliable and complete video processing.

Critical Fix: Face restoration now properly outputs ALL processed frames instead of just the first few frames, ensuring complete video reconstruction.

Critical Bug Fixes

  • Video Reassembly Failure: Fixed critical issue where face restoration produced videos with only 3 frames instead of the complete frame sequence (e.g., 1,257 frames → 3 frames)
  • Frame Loss During Extraction: Eliminated frame loss by removing FPS filtering during extraction, ensuring ALL original frames are preserved
  • Duration Preservation: Fixed video duration and timing issues by switching to duration-preservation video creation method
  • Frame Count Verification: Added frame count verification to ensure output video matches input frame count

Session Management & Organization

  • Numbered Session Folders: Face restoration now creates organized numbered session folders (e.g., "0003") similar to regular upscaling workflow
  • Organized Frame Storage: Separate subdirectories for extracted frames and processed frames within each session folder
  • Live Frame Saving: Frames are now saved to permanent storage immediately during processing, allowing users to monitor progress in real-time
  • Session Information Display: Enhanced UI to show session folder information, extracted frames location, and processed frames location

Enhanced Error Handling & Fallbacks

  • No-Face Fallback: Frames without detected faces are now saved as original images instead of causing processing failures
  • Face Restoration Fallback: If face restoration fails on a frame, the system automatically falls back to saving the original frame
  • Comprehensive Error Recovery: Enhanced error recovery mechanisms ensure batch processing continues even when individual frames encounter issues
  • Better Temp File Management: Improved temporary file cleanup and management to prevent storage issues during long processing sessions

Technical Improvements

  • Duration Preservation Method: Switched from `_reassemble_video_from_frames` to `create_video_from_frames_with_duration_preservation` for accurate video reconstruction
  • Enhanced Audio Merging: Improved audio preservation from original videos with better error handling for videos with/without audio tracks
  • Frame Extraction Optimization: Removed FPS filtering during frame extraction to preserve every single frame from the original video
  • Enhanced Logging: Comprehensive logging throughout the face restoration process with detailed progress tracking and verification

User Experience Improvements

  • Real-time Progress Monitoring: Users can now see extracted and processed frames in real-time during face restoration processing
  • Organized Output Structure: Consistent session-based folder organization makes it easier to locate and manage face restoration results
  • Progress Indicators: Enhanced progress reporting with detailed information about frames processed vs. frames with faces detected vs. original frames preserved
  • Session Summary: Comprehensive session information displayed in the UI showing folder locations and processing statistics
Important: This update fixes a critical bug that was causing face restoration to produce incomplete videos. All previous face restoration results should be re-processed with this version for complete video reconstruction.
# Technical Details: Video Reassembly Fix # Previous (buggy) method: _reassemble_video_from_frames() # Only processed ~3 frames # New (fixed) method: create_video_from_frames_with_duration_preservation() # Processes ALL frames

Version 3.0

Batch Processing Error Resolution & Stability Improvements

v3.0

Batch Processing Error Resolution & Enhanced Stability

July 2025

Version 3.0 focuses on resolving critical batch processing errors that were preventing successful completion of multi-video workflows. This release addresses various edge cases, error handling issues, and stability problems that could cause batch operations to fail or hang indefinitely.

Major Fix: Batch processing now handles errors gracefully and continues processing remaining videos in the queue without stopping the entire operation.

Critical Bug Fixes

  • Batch Queue Hanging: Fixed issue where batch processing would hang indefinitely when encountering corrupted or unsupported video files
  • Memory Leak in Batch Mode: Resolved memory accumulation during long batch processing sessions that could cause system crashes
  • Error Propagation: Fixed cascading errors where one failed video would cause subsequent videos in the batch to fail
  • Progress Tracking Issues: Corrected batch progress reporting that would show incorrect completion percentages
  • Temp File Cleanup: Fixed incomplete cleanup of temporary files during batch processing failures

Stability Improvements

  • Error Recovery: Enhanced error recovery mechanisms that allow batch processing to continue after individual video failures
  • Resource Management: Better GPU memory management during batch operations to prevent out-of-memory errors
  • File Validation: Added pre-processing validation to identify problematic files before starting batch operations
  • Logging Enhancement: Improved error logging and reporting for better debugging of batch processing issues

New Features

  • Batch Error Summary: Added comprehensive error summary report at the end of batch processing sessions
  • Skip Failed Videos Option: New option to automatically skip videos that fail initial validation checks
  • Retry Mechanism: Automatic retry functionality for transient errors during batch processing

Technical Changes

  • Exception Handling: Improved exception handling throughout the batch processing pipeline
  • Resource Cleanup: Enhanced automatic cleanup of resources and temporary files on error conditions
  • Thread Safety: Improved thread safety in batch processing operations to prevent race conditions
  • Status Monitoring: Enhanced real-time status monitoring and progress tracking for batch operations
Note: This update maintains full backward compatibility. Existing batch processing configurations will continue to work with improved reliability and error handling.

Version 2.0

Enhanced MKV Support & Improved Video Parsing

v2.0

Enhanced MKV Support & Robust Video Parsing

July 2025

Version 2.0 focuses on improving video file compatibility, particularly for MKV files, and enhancing the overall robustness of video parsing throughout the application. This release addresses issues where MKV files were not being properly recognized and parsed, leading to incorrect frame counts and duration information.

Major Improvement: MKV files now parse correctly with accurate frame counts, duration, and metadata extraction.

New Features

  • Enhanced MKV Support: Improved parsing for Matroska video files with proper handling of 'N/A' values in metadata fields
  • Robust Video Info Extraction: Multiple fallback mechanisms for video parsing when primary methods fail
  • OpenCV Integration: Automatic fallback to OpenCV for video information when ffprobe encounters parsing issues

Improvements

  • Video Parsing Robustness: Enhanced error handling for duration, frame count, and bitrate parsing from ffprobe output
  • Duration Calculation: Improved accuracy of video duration calculation using multiple methods and cross-validation
  • Frame Count Detection: Better frame count detection with fallback to direct frame counting when needed
  • Error Reporting: More detailed logging and error messages for video parsing issues

Bug Fixes

  • MKV Duration Parsing: Fixed "could not convert string to float: 'N/A'" error when parsing MKV file durations
  • Zero Frame Count Issue: Resolved issue where MKV files showed 0 frames and 0.000s duration
  • Bitrate Parsing: Fixed parsing errors for video bitrate information in various container formats
  • Format Detection: Improved format detection and handling for edge cases in video containers

Technical Changes

  • Enhanced get_video_info_fast(): Added 'N/A' value handling and improved error recovery
  • Enhanced get_video_info(): Improved comprehensive video information extraction with better fallbacks
  • OpenCV Fallback Integration: Automatic fallback to OpenCV for missing video information
  • Duration Calculation Logic: Enhanced duration calculation using frame count and FPS when direct parsing fails
Note: This update maintains full backward compatibility. All existing functionality remains unchanged, with only improvements to video parsing robustness.
